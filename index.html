Løsningsforslag

En webapp som oppfører seg som en app – uten appbutikk.

Første versjon kan være en responsiv nettside publisert med delbar URL. Etter hvert kan den bygges ut som PWA, slik at foreldre kan lagre den på mobilens hjemskjerm og få rask tilgang til siste oppdaterte informasjon.

Realiseringsplan

Foreslått vei fra testside til lagplattform.

  1. Avklare informasjonsmodell: hvilke felt trengs for trening, kamp, dugnad, turnering og beskjed?
  2. Lage MVP: statisk webapp med manuell redigering av innhold, publisert på GitHub Pages.
  3. Teste med Roterud G2014: 2–4 uker med foreldre, lagledere og trenere. Samle forbedringsønsker.
  4. Legge til enkel administrasjon: data i JSON, Google Sheets, Sanity, Supabase eller annet lett CMS.
  5. Gjøre den skalerbar: støtte flere lag, roller, filtrering og gjenbrukbare maler.
  6. PWA og varsling: hjemskjermikon, offline-cache og eventuelt push/e-post-varsler senere.

Delbar test-URL

Dette trengs i GitHub for å vise testsiden.

1. Repository

Legg denne koden i et GitHub-repository, for eksempel foreldreportal.

2. GitHub Pages

Aktiver Pages fra branch main eller work, mappe /root.

3. Delbar lenke

Etter publisering får dere en URL som kan deles med foreldre og trenere for test.

4. Videre utvikling

Opprett issues for ønsker: kalenderimport, påmelding, roller, admin og flere lag.
